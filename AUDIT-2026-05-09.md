# robOS — Audit Comprehensiv 2026-05-09

## TL;DR

robOS v2.0.0 are **fundatii solide** (hook architecture reala, atomic writes la file critice, multi-client routing testat in 45 assertii, license check Ed25519 fara confusion attacks, smoke tests verzi pe toate suprafetele) **dar are 2 BUG-uri CRITICE care silentios distrug functii promise** + **3 BLOCKER de onboarding** care, neatinse, vor face studentii sa abandoneze in primele 30 minute.

Cele mai grele: hook-urile nu citesc `.env` deloc (toate toggle-urile documentate sunt moarte), `session-timeout-detector` e orb pe per-client memory (fiecare sesiune client e fals-marcata abandonata), si Node version inconsistency intre setup-uri (20 vs 22.12).

**Verdict:** *nu trimite v2.0.0 catre studenti pana cand cele 5 finding-uri din "Action Plan" sunt rezolvate.* Restul (~80 finding-uri) sunt MED/LOW si pot fi escheduled.

**Cele 5 lucruri care daca le repari INAINTE sa trimiti v2.0.0 catre studenti:**

1. **F1 — Hooks citesc `.env`** (CRITICAL). Adauga `loadEnv()` la inceput in fiecare hook script. Smoke test ca `ROBOS_LOOP_DETECTOR_DISABLED=1` chiar dezactiveaza.
2. **F2 — session-timeout-detector multi-client** (CRITICAL). Inlocuieste hardcoded `MEMORY_DIR` cu walk-ul din `audit-startup.js:46-63`.
3. **U3 + D10 — Node version aliniat la 22.12.0** (BLOCKER). README:34 zice ">= 20", `setup.cmd:6` zice ">= 20", `setup.ps1:11` zice ">= 20", dar `setup.js:47` hard-fail-uieste pe < 22.12. Single-source MIN_NODE.
4. **U2 — sterge `docs/init/README.md`** (BLOCKER). E pentru produs DIFERIT ("Agentic OS" by Simon Scrapes / Agentic Academy). Reference `localhost:3000`, `mkt-brand-voice`, `bash scripts/centre.sh` care nu exista in robOS.
5. **F7 — Origin null bypass** (HIGH security). `centre/lib/auth.js:68-99` — `isSameOrigin` returns `true` cand Origin lipseste. Orice proces local (npm dep malicios, vscode ext) → fetch token → RCE prin skill-run.

- **CRITICAL F1**: Hooks IGNORA toate knob-urile din `.env`. Claude Code spawneaza hooks cu env curat. Operator seteaza `ROBOS_CHECKPOINT_DISABLED=1`, nu se intampla nimic. Singurul knob care merge: `ROBOS_DEV` (license-check.js are propriul parser inline). Sursa: `scripts/checkpoint-reminder.js:32,79`, `scripts/activity-capture.js:158`, `scripts/note-candidates.js:190`, `scripts/lib/loop-detector.js:194` — zero `loadEnv()` calls.
- **HIGH F3**: 3 politici diferite pentru memory directory: `audit-startup` walks toate scope-urile, `checkpoint-reminder` walks doar active client, `session-timeout-detector` walks doar root. State drift intre hooks.
- **HIGH F7**: Dashboard `/api/auth/token` accepta requests fara Origin header (returneaza true in `isSameOrigin`). Orice CLI / npm dep / vscode ext local poate fura token-ul fara browser → CSRF claim din AGENTS.md fals.
- **HIGH S3**: `ARGS_FORBIDDEN_RE` din `centre/api/system.js:278` blocheaza spatii → orice input multi-cuvant catre `/api/skills/*/run` returneaza 400. Feature spart silent.

### 3. Multi-client edge cases

- **CRITICAL F2**: `session-timeout-detector.js:34` hardcoded `MEMORY_DIR = ROBOS_ROOT/context/memory`. Cand client activ → memory scrisa in `clients/{slug}/context/memory/` → cron crede ca toate sunt abandonate → false recovery flags + sterge `{sid}.json` markers (linia 129) → corupe checkpoint la urmatoarea sesiune. **Fiecare client real va declansa fals alarme**.
- **HIGH F3**: Memory directory inconsistency intre hooks (vezi #1).
- **LOW F19**: `clients/` e gol pe disk. End-to-end "add client + onboard + brand voice in client folder" niciodata exersat in productie. Smoke acopera lib + CLI, dar nu fluxul real.

### 4. Stability / failure modes

- **HIGH F4**: Atomic write *.tmp leak pe Windows. Confirmat: `data/session-state/c012d60a-...-tools.json.9891eb88.tmp` ramas dupa EBUSY/EPERM. `loop-detector.js:62-68` si `client-context.js:45-51` nu au try/finally cleanup. Counter-ul de loop e silently stale dupa crash.
- **MED F9**: NDJSON rotation race window. `ndjson-log.js:74-94` foloseste `path + '.tmp'` non-randomizat. Daca rotation A si B concurente sau un append intre `readFileSync` si `renameSync` → linia noua se pierde.
- **MED F14**: `cron-leader-lock.js:107-130` — heartbeat write nu atomic. Crash mid-write → lock corupt → 2 leaderi → 2× cron jobs (token waste + duplicat side-effects).
- **MED F18**: `centre/api/settings.js:setEnv` nu retry pe Windows file lock. Operator deschide `.env` in editor → dashboard scrie → renameSync EPERM → silent failure, `.env.tmp` orfan.
- **MED S1**: `.env.bak` creat fara `mode: 0o600` explicit. POSIX OK daca `.env` deja restrictionat; Windows: ACL inheritance.
- 6 finding-uri LOW.

### 5. Scalability / aging

- **HIGH F5**: `data/session-state/` zero retention. 69 fisiere acumulate (33 markers, 20 checkpoints, 2 tools.json + smoke leftovers). La 1-3 sesiuni/zi → 1000+ fisiere/an. `session-timeout-detector` face `readdirSync` peste tot la fiecare 15 min → O(n) syscalls Windows.
- **MED F10**: `data/session-recovery/` la fel — niciodata sterse, marcate `consumed:true` si lasate. Linear-growth read la fiecare cold-session start.
- **MED F11**: SQLite migrations — `runMigrations` swallows "duplicate column" si tot face `INSERT OR REPLACE INTO schema_version`. Daca o migratie partial-failed e inghitita → schema_version bumped fara invariantul respectat → silent drift la upgrade.
- **MED F12**: `parallel-budget.js:64-67` foloseste `appendFileSync` direct, fara rotation cap. Daca skills incep sa logheze telemetria asa cum spune AGENTS.md, fisierul creste nelimitat.
- **LOW**: `context/memory/*.md` cresc forever. La 1 an = ~365 fisiere. Doar audit-startup ultimele 7 zile filtreaza, dar disk usage si dashboard listing nu au strategie.

### 6. Security & secrets

- **HIGH F7**: Origin null bypass (#1, repetat aici pentru categoria security).
- **HIGH S3**: ARGS_FORBIDDEN_RE blocheaza spatii (#1). Bug functional in security control.
- **HIGH S4**: `cron-runner.js:107` foloseste `shell: true` pentru job-uri cu camp `command`. API blocheaza setarea, dar skill malicios poate scrie `cron/defaults/*.json` → RCE prin shell metacharacters. Defense-in-depth defect.
- **MED S5**: Activity log redactie best-effort. Pattern-uri `KEY=value` cu nume custom (`STRIPE_RESTRICTED_KEY=rk_live_xxx`) scapa. `redact.js` acopera doar prefixele cunoscute (sk-, sk-ant-, ghp_).
- **MED S7**: `ROBOS_CENTRE_HOST=0.0.0.0` override printeaza "loopback only" misleading la startup. Student care expune in LAN nu vede warning.
- **MED S8/S9**: `GET /api/files` si `GET /api/system/activity` NU sunt sub Bearer auth. CSRF read potential prin browser malicios (CORS protejaza partial, dar nu deplin — extensii cu host permissions bypass).

## Strengths (lucruri pe care robOS le face BINE)

1. **Hook error sink + "exit 0 always"** — broken hook NICIODATA blocheaza prompt-ul user-ului. Tested live: malformed stdin → silent exit 0. #1 lucru bine facut.
2. **Session ID sanitization** consistent (regex `^[a-zA-Z0-9_-]{1,128}$`) in toate hooks. Path-traversal via session_id genuinely blocked.
3. **Ed25519 license JWT** — `cryptoVerify(null, ...)` cu key type → algorithm confusion (`alg: none`, RS256→HS256) imposibila structural.
4. **Slug validation** stricta + self-healing pe client folder lipsa. 45 assertii smoke verzi.
5. **Loop detector** design solid (canonical JSON hashing, threshold + escalation, exempt list, env-disable [DAR vezi F1!], corrupt state recovery). 69 assertii smoke verzi.
6. **Bearer auth + timing-safe compare** in `centre/lib/auth.js:28` — corect implementat, doar Origin null bypass-ul (F7) e gap-ul.
7. **Atomic writes** in env-format, client-context, loop-detector — pattern temp+rename consistent (cu reservele F4).
8. **Parameterized SQL** universal in licensing/D1 layer — zero SQL injection surface.
9. **`docs/operator-handbook.md` + `docs/cheat-sheet.md`** — exista si sunt student-friendly (dar nu linked vizibil din top-level).
10. **License check graceful degradation** — daca verify crashes, falls open rather than DoS-ing paying user. Threat-modeled, nu thrown together.
